Each agent is described by two groups of metadata — model-level and scaffold-level. These features feed into a projection network fagent(·) that provides a cold-start prior for the latent ability vector θ, even for agents never seen during training.

Latent ability is a sum of a learned embedding (seen agents) and a feature-based projection. For novel agents, only the projection is available — this is the cold-start prior.
P(yij=1) = σ(θi·aj + dj)
Inner product of agent ability and task discrimination determines difficulty. K controls the number of latent dimensions — K=2 is recommended for interpretability without sacrificing fit.
Loss = −log P(yij) + λ‖θ‖²
Binary cross-entropy over observed response matrix entries, with L2 regularisation on agent embeddings to prevent overfitting on sparse agents.
K-factor sweep — val AUC ?Why AUC?Area Under the ROC Curve measures rank-order discrimination: does the model correctly rank a harder agent below an easier one on a given task? AUC = 0.5 is random; AUC = 1.0 is perfect ordering. It is threshold-agnostic — useful when we care about relative rankings across 273 agents more than a single cut-point. & calibration error ?Why calibration?A calibrated model is right about its confidence: when it predicts 70% pass rate, ~70% of agents actually pass. Poor calibration inflates predicted probabilities for hard tasks or deflates them for easy ones, making adaptive task selection less reliable. We target cal err < 0.05 — below that, the probability estimates are trustworthy enough to drive task-subset decisions. (features on, K ∈ {1,2,4,8})

Holistic Agent Leaderboard: The Missing Infrastructure for AI Agent Evaluation
Sayash Kapoor, Benedikt Stroebl, Peter Kirgis et al. · 2025
Introduces a standardized harness and public leaderboard that orchestrates parallel agent evaluations across hundreds of VMs, covering coding, web navigation, and customer service tasks.
This work builds directly on the public HAL leaderboard dataset — the response matrix of 13 harnesses × 36 models × 1796 tasks across 8 benchmarks is extracted from HAL's open evaluation results. HAL provides the infrastructure; we apply IRT on top to make that data actionable for adaptive evaluation.
arxiv.org/abs/2510.11977 ↗

Reliable and Efficient Amortized Model-based Evaluation
Sang Truong, Yuheng Tu, Percy Liang, Bo Li, Sanmi Koyejo · 2025
Uses item response theory with trained difficulty predictors and adaptive question generation to enable reliable LM evaluation at a fraction of the cost across multiple benchmarks.
We draw on the idea of learning a latent capability embedding per agent — the MIRT θ vector plays the same role as the amortised model representation here, letting us predict performance on unseen tasks from a compact learned prior rather than exhaustive evaluation.
arxiv.org/abs/2503.13335 ↗
[2]
tinyBenchmarks: evaluating LLMs with fewer examples
Felipe Maia Polo, Lucas Weber, Leshem Choshen, Yuekai Sun, Gongjun Xu, Mikhail Yurochkin · 2024
Demonstrates that LLM benchmark performance can be accurately estimated from ~100 curated IRT-selected examples rather than full datasets of thousands, releasing compact versions of popular benchmarks.
We directly extend the CAT (Computerised Adaptive Testing) principle to agent benchmarks: rank tasks by fitted IRT discrimination ‖aj‖ and keep only the high-signal subset. Where tinyBenchmarks targets LLM eval on fixed QA sets, we apply the same cost-reduction logic to longer-horizon agentic tasks across 8 benchmarks, achieving 25–44% task reduction.
arxiv.org/abs/2402.14992 ↗
